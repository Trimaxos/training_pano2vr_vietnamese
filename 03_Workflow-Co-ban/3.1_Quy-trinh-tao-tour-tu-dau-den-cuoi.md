# 3.1 – Quy trình tạo tour từ đầu đến cuối

## 1. Mục tiêu bài học

- Sau bài này, học viên sẽ:
  - Nắm được **5 bước lớn** của một quy trình tạo tour web chuẩn.
  - Biết đầu ra (deliverable) của mỗi bước là gì.
  - Có bức tranh tổng thể trước khi đi sâu vào từng bài tiếp theo.

## 2. Tổng quan quy trình

```
[1. Chuẩn bị ảnh đầu vào]
         ↓
[2. Tạo project & import node]
         ↓
[3. Cấu hình node (Properties, User Data, Patching)]
         ↓
[4. Thêm Hotspot & liên kết tour]
         ↓
[5. Thiết kế Skin (giao diện)]
         ↓
[6. Export Web & kiểm tra]
```

Phần 3 này tập trung vào bước 1 → 3. Hotspot (bước 4) học ở Phần 4, Skin (bước 5) ở Phần 5, Export chi tiết (bước 6) ở Phần 6.

## 3. Bước 1 – Chuẩn bị ảnh đầu vào

### 3.1.1. Yêu cầu ảnh

| Tiêu chí | Yêu cầu |
|---|---|
| Định dạng | JPEG, PNG, TIFF, WebP |
| Tỷ lệ | 2:1 (equirectangular) – VD: 8000×4000, 12000×6000 |
| Chất lượng | Tối thiểu 6000×3000px cho tour web chất lượng cao |
| Đã stitch | Pano2VR không stitch – ảnh phải được stitch bằng PTGui, Hugin, RealityCapture hoặc phần mềm camera |

### 3.1.2. Tổ chức file trước khi import

Khuyến nghị cấu trúc folder:

```
ProjectName/
├── _source/           ← Ảnh gốc (không xử lý)
│   ├── Tang-1/
│   └── Tang-2/
├── _patched/          ← Ảnh sau khi patch/xử lý
├── _output/           ← Thư mục xuất output web
└── ProjectName.p2vr   ← File project Pano2VR
```

> Đặt folder `_source` và `_patched` cùng nơi với file `.p2vr` giúp tránh lỗi "Missing Assets" khi di chuyển project.

## 4. Bước 2 – Tạo project & import node

1. Mở Pano2VR → **New Project** (`Ctrl+N`).
2. **Save** ngay (`Ctrl+S`) vào folder project (VD: `D:\Projects\VinTour\VinTour.p2vr`).
3. Kéo thả folder chứa ảnh vào **Tour Browser** (hoặc nút Add Input).
4. Pano2VR tự tạo node cho mỗi ảnh, folder name → tag tự động.
5. Đổi tên node nếu cần (Properties → User Data → Title).

## 5. Bước 3 – Cấu hình node

Làm lần lượt cho từng node:

1. Chọn node trong Tour Browser.
2. Kiểm tra **Input Type** (Equirectangular?).
3. Điền **Title** (tên phòng/ vị trí).
4. Căn chỉnh **North Offset** (nếu cần).
5. Đặt **Initial View** (nhấn `I` sau khi chỉnh góc nhìn đẹp).
6. Xem ảnh trong Viewer – nếu có vùng cần **Patch** (nadir, nadirlogo, ống kính,…) → làm Patching (xem bài 3.3).
7. Lặp lại cho node tiếp theo.

> Mẹo: Dùng **Copy Node Settings** để copy Initial View settings từ 1 node sang nhiều node cùng loại (VD: tất cả phòng ngủ cùng FoV 90°, tilt 0°).

## 6. Bước 4, 5, 6 – Sơ lược

| Bước | Nội dung | Bài học |
|---|---|---|
| 4. Hotspot | Thêm điểm dẫn chuyển cảnh, info popup, link URL | Phần 4 |
| 5. Skin | Thiết kế menu, nút, logo, thumbnails | Phần 5 |
| 6. Export | Cấu hình & xuất Web HTML5, kiểm tra | Phần 6 |

## 7. Deliverable của mỗi bước

| Bước | Đầu ra |
|---|---|
| 1 | Folder ảnh gốc đã tổ chức sẵn |
| 2 | File `.p2vr` với đủ node, tên rõ ràng |
| 3 | Tất cả node đã có Title, Initial View, không còn vùng lỗi |
| 4 | Tour có thể duyệt từ node này sang node khác qua hotspot |
| 5 | Tour có giao diện đầy đủ (logo, menu, thumbnail) |
| 6 | Folder web hoàn chỉnh, chạy được trên browser |

## 8. Tài liệu tham khảo

- Getting Started – `https://ggnome.com/doc/pano2vr/guide/getting-started/`

## 9. Gợi ý bài tập

1. Lên kế hoạch cho một **tour thực tế nhỏ** (5–10 node): vẽ sơ đồ liên kết các phòng/vị trí.
2. Tổ chức folder theo cấu trúc khuyến nghị trên – import ảnh vào project mới.
3. Điền đầy đủ Title cho tất cả node trước khi sang bài tiếp theo.
