# 3.8 – Local server và test trên browser

## 1. Mục tiêu bài học

- Sau bài này, học viên sẽ:
  - Hiểu vì sao output web cần chạy qua local server.
  - Biết test tour trên browser đúng cách.
  - Biết các lỗi thường gặp khi mở sai cách.

## 2. Vì sao không nên mở `index.html` trực tiếp?

- Khi mở bằng `file://`, nhiều trình duyệt sẽ chặn một số request nội bộ vì lý do bảo mật.
- Hậu quả thường gặp:
  - Tour không load hết.
  - Tile không hiện.
  - Skin/script chạy lỗi.

Do đó, output web nên được test qua:
- Local server của Pano2VR.
- Hoặc một web server cục bộ khác.

## 3. Cách test bằng local preview của Pano2VR

1. Tạo `Web Output`.
2. Generate output.
3. Dùng nút `Preview` của output đó.
4. Pano2VR sẽ mở browser với địa chỉ local dạng:
   - `http://localhost:8080/...`

## 4. Những gì cần test trên browser

- Loading ban đầu.
- Chất lượng hình ảnh.
- Hotspot click được không.
- Menu và skin hoạt động đúng không.
- Resize cửa sổ có bị vỡ layout không.

## 5. Nên test trên browser nào?

Tối thiểu:
- Chrome
- Edge

Nếu dự án yêu cầu cao hơn:
- Firefox
- Mobile browser

## 6. Test mobile

Nếu có thể, nên test trên điện thoại thật để kiểm tra:
- Cảm giác vuốt xoay.
- UI có quá nhỏ không.
- Tốc độ tải.
- Gyroscope nếu có bật.

## 7. Các lỗi thường gặp

| Lỗi | Nguyên nhân |
|---|---|
| Output mở trắng trang | Mở sai bằng `file://` hoặc lỗi đường dẫn |
| Hotspot không phản hồi | Link/action chưa đúng hoặc script lỗi |
| Hình tải chậm | Ảnh quá nặng hoặc tile chưa tối ưu |
| UI lệch trên mobile | Skin chưa thiết kế responsive |

## 8. Mẹo thực tế

- Dùng tên folder output ngắn, không dấu.
- Nếu cổng preview bị chiếm, đổi trong `Preferences`.
- Mỗi lần generate lại, refresh browser để kiểm tra đúng bản mới nhất.

## 9. Tài liệu tham khảo

- Preferences – `https://ggnome.com/doc/pano2vr/workspace/preferences/`
- Web Output – `https://ggnome.com/doc/pano2vr/output/web/`

## 10. Gợi ý bài tập

1. Generate một bản Web Output.
2. Mở bằng preview local server.
3. Kiểm tra trên ít nhất 2 browser khác nhau.
4. Nếu có điều kiện, mở thử trên điện thoại cùng mạng.
