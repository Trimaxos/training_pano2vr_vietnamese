# 3.2 – Chuẩn bị ảnh đầu vào

## 1. Mục tiêu bài học

- Sau bài này, học viên sẽ:
  - Biết tiêu chuẩn tối thiểu của ảnh panorama trước khi đưa vào Pano2VR.
  - Biết cách tổ chức folder nguồn để tránh lỗi missing files.
  - Phân biệt ảnh dùng trực tiếp được với ảnh cần xử lý trước.

## 2. Pano2VR nhận loại input nào?

- Pano2VR có thể làm việc với:
  - Ảnh panorama `equirectangular` 2:1.
  - Một số input đặc biệt như `cubemap`, `fisheye`, `flat image`, `360 video`.
- Trong workflow phổ biến nhất, đội triển khai sẽ dùng:
  - `JPEG` panorama 2:1 đã stitch hoàn chỉnh.

> Pano2VR không phải phần mềm stitch. Nếu ảnh chưa stitch, cần xử lý trước bằng phần mềm khác.

## 3. Tiêu chuẩn ảnh đầu vào

| Hạng mục | Khuyến nghị |
|---|---|
| Tỷ lệ ảnh | 2:1 |
| Độ phân giải tối thiểu | 6000×3000 |
| Khuyến nghị tốt | 8000×4000 đến 12000×6000 |
| Định dạng | JPG chất lượng cao hoặc PNG |
| Màu sắc | Đồng nhất giữa các node |
| Metadata GPS | Có thì tốt, không bắt buộc |

## 4. Những lỗi thường gặp trước khi import

| Lỗi | Hậu quả |
|---|---|
| Ảnh không đúng tỷ lệ 2:1 | Viewer hiển thị méo hoặc nhận sai projection |
| Ảnh quá nặng, chưa tối ưu | Project lag, generate lâu |
| Tên file lộn xộn | Khó quản lý node sau khi import |
| Ảnh chưa retouch nadir | Sau này phải patch lại nhiều |
| Ánh sáng lệch mạnh giữa các pano | Chuyển cảnh thiếu tự nhiên |

## 5. Tổ chức thư mục nguồn

Khuyến nghị:

```text
DuAn_Tour/
├── _source/
│   ├── Tang-1/
│   ├── Tang-2/
│   └── Ngoai-that/
├── _patched/
├── _output/
└── DuAn_Tour.p2vr
```

- Đặt tên folder theo khu vực để tận dụng auto tag khi import vào Tour Browser.
- Hạn chế dùng tên file quá dài hoặc ký tự đặc biệt.
- Ưu tiên tên không dấu nếu dự án còn phải upload web hoặc xử lý bằng script.

## 6. Quy ước đặt tên nên dùng

Ví dụ:

- `tang1-sanh-chinh.jpg`
- `tang1-phong-khach.jpg`
- `tang2-hanh-lang-01.jpg`
- `ngoai-that-cong-chinh.jpg`

Tên tốt cần:
- Dễ hiểu với người trong team.
- Phản ánh vị trí thực tế.
- Có logic để sort.

## 7. Kiểm tra trước khi import

Checklist nhanh:

1. Ảnh đã stitch xong.
2. Tỷ lệ đúng 2:1.
3. Không còn lỗi ghép rõ rệt.
4. Nếu có logo/ tripod ở nadir, đã xác định có patch hay retouch sau.
5. Ảnh được đặt đúng folder theo khu vực.

## 8. Tài liệu tham khảo

- Project Setup – `https://ggnome.com/doc/pano2vr/guide/getting-started/project-setup/`
- Input Images – `https://ggnome.com/doc/pano2vr/reference/input-images/`

## 9. Gợi ý bài tập

1. Chuẩn bị một bộ 5–10 panorama và tổ chức lại theo cấu trúc folder chuẩn.
2. Đổi tên toàn bộ file theo quy ước thống nhất.
3. Kiểm tra từng ảnh xem có node nào cần patch trước khi dựng tour hay không.
