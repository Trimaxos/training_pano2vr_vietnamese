# 6.6 – Cloud, Package và triển khai

## 1. Mục tiêu bài học

- Sau bài này, học viên sẽ:
  - Hiểu các hướng triển khai tour sau khi export.
  - Biết khi nào nên dùng cloud, khi nào nên tự host.

## 2. Các hướng triển khai phổ biến

| Hướng | Mô tả |
|---|---|
| Local preview | Kiểm tra nội bộ trên máy |
| Hosting riêng | Upload lên web server của công ty/khách hàng |
| Cloud của Ggnome | Chia sẻ nhanh, ít cấu hình |
| Package | Đóng gói để bàn giao hoặc upload |

## 3. Khi nào nên dùng cloud?

- Cần gửi demo nhanh.
- Không muốn cấu hình hosting phức tạp.
- Cần chia sẻ cho khách hàng review sớm.

## 4. Khi nào nên tự host?

- Cần kiểm soát domain riêng.
- Cần tích hợp với website có sẵn.
- Cần chính sách bảo mật hoặc hạ tầng riêng.

## 5. Checklist trước khi triển khai

- Output generate không lỗi.
- Tên folder/file sạch.
- Test trên browser ổn.
- Các asset đi kèm đầy đủ.
