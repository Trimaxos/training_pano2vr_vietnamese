# 1.2 – Cài đặt và khởi động Pano2VR

## 1. Mục tiêu bài học

- Sau bài này, học viên sẽ:
  - Nắm được **yêu cầu hệ thống cơ bản** để chạy Pano2VR 7.1.
  - Biết cách **tải và cài đặt** Pano2VR đúng phiên bản, dùng thử hoặc nhập license.
  - Mở được Pano2VR, hiểu nhanh **màn hình chào (Welcome)** và **các cách tạo project mới**.
  - Biết chuẩn bị **thư mục dự án** theo đúng workflow trong tài liệu training này.

## 2. Yêu cầu hệ thống & chuẩn bị trước

### 2.1. Hệ điều hành & cấu hình cơ bản

(Tham khảo docs “Installation” & FAQ system requirements)

- **Hệ điều hành**:
  - **Windows**: Windows 10 (1809 trở lên) hoặc Windows 11.
  - **macOS**: từ **macOS 11** trở lên (theo docs mới).
- **RAM**: tối thiểu 4 GB, nên 8 GB+ cho tour lớn/ ảnh gigapixel.
- **GPU**: card hỗ trợ OpenGL (đa số máy hiện nay đều đáp ứng; nếu Viewer giật/ đen màn, nên cập nhật driver).

> Gợi ý training: với máy dùng để dựng tour sản xuất cho khách, nên test trước 1–2 dự án lớn để xem máy có đủ khoẻ không (zoom, xoay, preview web mượt).

### 2.2. Chuẩn bị ảnh/ video 360 (rất quan trọng)

- **Pano2VR không stitch ảnh**.  
  - Bạn cần dùng phần mềm chuyên stitch (VD: PTGui, Hugin, Mistika VR, v.v.) để tạo file 360° trước.
- **Định dạng khuyến nghị** cho input:
  - Ảnh **equirectangular** tỉ lệ **2:1** (VD: 12000×6000, 8000×4000, 6000×3000,…).  
  - Với video: file 360° đã stitch sẵn (thường MP4/H.264 hoặc H.265) ở chế độ equirectangular.
- Những lỗi thường gặp nếu input không đúng:
  - Ảnh bị kéo méo, seam lộ rõ, đường chân trời nghiêng.  
  - Viewer xoay bị “say” vì ảnh không được level.  
  - Không dùng được một số tính năng (ví dụ map/ GPS, auto‑link) nếu thiếu metadata.

Trong các bài sau (Phần 3), bạn sẽ học cách **leveling, set north, patching** bên trong Pano2VR sau khi đã import media đúng chuẩn.

## 3. Tải và cài đặt Pano2VR

### 3.1. Tải đúng phiên bản

- Trang chính product: `https://ggnome.com/pano2vr/`  
  - Phần **Download** thường có:
    - **Stable Version** (VD: 7.1.11),
    - **Beta Version** (VD: 8.0 beta).
- Với tài liệu training này, **ưu tiên bản stable 7.1.x** (hoặc bản stable gần nhất tương đương).

### 3.2. Các bước cài đặt cơ bản

Trên Windows (tương tự macOS, chỉ khác installer):

1. **Tải file cài đặt**:
   - `pano2vr_install64_7_1_xx.exe` (Windows)  
   - `pano2vr_macos_7_1_xx.dmg` (macOS)
2. **Chạy installer**:
   - Chọn thư mục cài đặt (thường để mặc định).  
   - Đồng ý license, Next → Finish.
3. Sau khi cài xong:
   - Trên Windows: tìm **Pano2VR** trong Start Menu.  
   - Trên macOS: kéo icon Pano2VR vào thư mục Applications (tuỳ hướng dẫn trên .dmg).

> Lưu ý cho môi trường công ty:  
> - Nếu máy bị hạn chế quyền cài phần mềm, cần IT hỗ trợ/ cài qua account admin.  
> - Nên thống nhất **thư mục cài đặt mặc định** để sau này dễ viết hướng dẫn chung.

### 3.3. Dùng thử vs nhập license

- Pano2VR cho phép **dùng thử**:
  - Giới hạn thường là số node tối đa, và/hoặc watermark trên output.  
  - Phù hợp cho training nội bộ, test nhanh feature.
- **Nhập license**:
  - Sau khi mua, bạn được cung cấp **license key** (qua email hoặc account Gnome).  
  - Trong Pano2VR:
    - Vào **Help → Register** (hoặc tương đương tuỳ OS).  
    - Dán key, xác nhận.  
  - Nên lưu lại key ở nơi an toàn (hoặc để IT quản lý).

## 4. Màn hình khởi động & cách tạo project mới

### 4.1. Màn hình Welcome (khái quát)

Khi mở Pano2VR lần đầu, thường sẽ có:

- **Danh sách project gần đây** (Recent Projects).  
- Nút **New Project** (dùng Project Wizard ở v8, hoặc workflow basic ở 7.1).  
- Nút mở project có sẵn (**Open Project**).  
- Đường dẫn nhanh tới **Getting Started** và **User Guide**.

Trong training này, ta chủ yếu dùng:

- **New Project** để tạo project mẫu theo workflow cơ bản.  
- **Open Project** để mở lại các ví dụ/ bài tập đã lưu.

### 4.2. Cấu trúc thư mục dự án đề xuất (theo training)

Để dễ quản lý và khớp với lộ trình tài liệu, đề xuất:

- Mỗi dự án (khách/ công trình) có 1 thư mục gốc, ví dụ:

  - `Dự-án_XYZ\`  
    - `input\` (ảnh/video 360 đã stitch)  
    - `pano2vr\` (file `.p2vr`, skin `.ggsk`, patch, asset phụ)  
    - `output_web\` (thư mục xuất HTML5 cho khách)  
    - `docs\` (tài liệu training/ bàn giao, nếu có)

- Đối với **bài tập training** trong thư mục `Tai_lieu_training`:
  - Bạn có thể tạo thêm 1 thư mục riêng, ví dụ:  
    - `Tai_lieu_training\Pano2VR\Demo_projects\`  
    - Mỗi bài demo 1 project nhỏ (VD: `01_basic-tour`, `02_hotspots`,…).

Mục tiêu:  
**Tách rõ** input gốc, file Pano2VR, và output web → sau này sửa/ xuất lại không nhầm lẫn.

### 4.3. Tạo project mới – workflow tối thiểu

Bước chi tiết sẽ học ở Phần 3, ở đây chỉ nêu khung:

1. Tạo thư mục project theo cấu trúc đề xuất.  
2. Mở Pano2VR → **New Project**.  
3. Chỉ định **Project file (.p2vr)** lưu trong thư mục `pano2vr\`.  
4. **Import input media** (kéo thả ảnh/ folder vào Viewer hoặc Tour Browser).  
5. Lưu project, sau đó mới bắt đầu các thao tác tiếp (set default view, patching, linking,…).

## 5. Lưu ý cho môi trường công ty / team

- **Quy ước phiên bản**:
  - Ghi rõ trong tài liệu nội bộ: đang chuẩn hoá theo **Pano2VR 7.1.x**.  
  - Nếu máy ai đó dùng 8.x beta, nên ghi chú khi share project/ output (vì có thể có khác biệt).

- **Quy ước nơi cài & nơi lưu project**:
  - Hạn chế lưu project chính thức vào `Desktop`/ `Downloads`.  
  - Nên dùng chung 1 cấu trúc thư mục mạng/ OneDrive/ SharePoint được backup.

- **Backup & update**:
  - Bật/ cấu hình tính năng **backup** trong Settings (sẽ nói kỹ ở phần Workspace/ Settings).  
  - Khi update phiên bản mới, nên:
    - Xem mục **What’s New** trước (để biết có gì thay đổi).  
    - Test một vài project mẫu trước khi cập nhật hàng loạt máy.

## 7. Gợi ý bài tập

1. **Tự cài Pano2VR** trên máy của bạn (hoặc kiểm tra phiên bản đang có):  
   - Ghi lại: OS, phiên bản Pano2VR, nơi cài đặt, nơi lưu project mặc định.
2. **Tạo 1 project trống**:
   - Tạo thư mục `Demo_Pano2VR_Basic\` với các subfolder như gợi ý.  
   - Tạo file `.p2vr` và import ít nhất 2 ảnh 360 (có thể lấy từ sample `Building_a_tour.zip` trong docs).
3. Ghi chú lại **những điểm khác** giữa máy bạn và hướng dẫn (về đường dẫn, version, quyền cài đặt) để chuẩn hoá sau.
