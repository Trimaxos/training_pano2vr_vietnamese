# 2.2 – Tour Browser

## 1. Mục tiêu bài học

- Sau bài này, học viên sẽ:
  - Biết **Tour Browser là gì** và tại sao nó chỉ có ở Pano2VR Pro.
  - Thực hiện được các thao tác cơ bản: import node, đặt Start Node, đổi thứ tự, gán tag, xoá node.
  - Hiểu khái niệm **node** và **thumbnail** trong context Tour Browser.
  - Biết dùng **filter** để tìm node nhanh trong tour lớn.

## 2. Tour Browser là gì?

- **Tour Browser** là panel quản lý toàn bộ **node (cảnh panorama)** của tour.
- Mỗi node = 1 ảnh / 1 video 360° (input media) đã được import vào project.
- Hiển thị dưới dạng **thumbnail** (ảnh thu nhỏ), kèm tên node.
- Chỉ có trong **Pano2VR Pro** – đây là một trong những lý do chính để chọn Pro cho dự án tour thực tế.

Mở Tour Browser:
- Toolbar → biểu tượng Tour Browser, hoặc **Alt+Shift+T** (Windows).
- Hoặc menu **Window → Tour Browser**.

Docs tham khảo: `https://ggnome.com/doc/pano2vr/workspace/tour-browser/`

## 3. Các thao tác cơ bản

### 3.1. Import node vào Tour Browser

Có 3 cách thêm node:

1. **Kéo thả** file/ folder từ File Explorer thẳng vào Tour Browser hoặc Viewer.
   - Kéo cả folder → các node bên trong sẽ được tag tự động bằng tên folder.
2. **Nút Input** trên Toolbar → chọn file/ folder.
3. **Menu File → Import** (tuỳ phiên bản).

> Lưu ý: Pano2VR **không stitch**. File input phải là ảnh equirectangular 2:1 hoặc video 360° đã stitch.

### 3.2. Đặt Start Node (node bắt đầu)

- Node đầu tiên trong tour khi xuất web là **Start Node** (ký hiệu "1" hoặc icon home).
- Cách đặt: **chuột phải** vào node muốn làm Start → **Set as Start Node**.
- Hoặc kéo node đó lên vị trí đầu tiên trong danh sách.

### 3.3. Đổi thứ tự node

- **Kéo thả** thumbnail trong Tour Browser để sắp xếp lại thứ tự.
- Thứ tự ảnh hưởng tới:
  - Thứ tự trong menu thumbnail (nếu dùng skin có menu).
  - Thứ tự khi dùng Sequential Linking (liên kết tự động theo thứ tự).

### 3.4. Chọn node để làm việc

- **Click** vào thumbnail → node đó hiển thị trong Viewer, Properties cập nhật thông tin của node đó.
- **Di chuyển nhanh qua các node**: `Ctrl+]` (Next), `Ctrl+[` (Previous).

### 3.5. Xoá node

- Chuột phải vào node → **Remove from Tour** (xoá khỏi tour, không xoá file gốc trên ổ đĩa).

> Phân biệt: Remove from Tour ≠ xoá file. File input vẫn còn nguyên trên ổ đĩa.

### 3.6. Gán tag (Tag)

- Chuột phải vào node → **Add Tag**.
- Nhập tên tag (VD: "Tang-1", "Phong-khach", "Ngoai-that").
- Tag có thể dùng để:
  - **Lọc (filter)** node trong Tour Browser.
  - Tạo **menu phân nhóm** trong Skin Editor.
  - Dùng trong **Automatic Linking** theo vùng.

> Nếu kéo cả folder vào, folder name tự động trở thành tag cho toàn bộ node trong folder đó.

### 3.7. Lọc node (Filter)

- Phía trên Tour Browser có ô **Search/ Filter**:
  - Gõ tên node để tìm.
  - Lọc theo tag.
  - Lọc theo khoảng cách GPS (nếu node có metadata vị trí).
- Hữu ích khi tour có hàng chục đến hàng trăm node.

### 3.8. Context menu (chuột phải) – tóm tắt

| Tuỳ chọn | Ý nghĩa |
|---|---|
| Set as Start Node | Đặt làm node bắt đầu |
| Change Input Type | Đổi loại input (pano, video, flat,…) |
| Remove from Tour | Xoá khỏi tour (không xoá file) |
| Add Tag / Remove Tag | Gán/ bỏ tag |
| Sort nodes… | Sắp xếp theo tên, ngày, thứ tự import,… |

## 4. Node thumbnail & tooltip

- Hover chuột vào thumbnail → tooltip hiển thị:
  - Tên node.
  - Filename (đường dẫn file gốc).
  - Node ID (dùng khi tham chiếu trong skin/ logic/ API).
- Node ID được tạo tự động; **không thay đổi** node ID tùy tiện khi tour đang chạy production vì các hotspot/ skin đang trỏ đến ID cũ sẽ bị gãy.

## 5. Lưu ý thực tế

- **Tour lớn** (50+ node): đặt tên node rõ ràng ngay khi import (trong User Data → Title). Sửa tên sau rất mất thời gian.
- **Tag ngay từ đầu**: khi drag folder vào, folder name thành tag → tận dụng cách đặt tên folder để tag tự động (VD: folder "Tang-1", "Tang-2", "San-thuong").
- **Backup thứ tự**: nếu thứ tự node quan trọng (VD: tour theo tuyến), lưu file `.p2vr` thường xuyên và đừng lỡ tay sort.
- **Missing assets**: nếu file gốc bị di chuyển/ đổi tên, thumbnail sẽ hiển thị icon lỗi. Dùng **Find and Relocate Missing Files** (menu File) để sửa.

## 7. Gợi ý bài tập

1. Tạo 1 project mới, import **ít nhất 5 ảnh** 360° từ 2 folder khác nhau.
   - Xác nhận: mỗi folder có tag đúng tên folder.
   - Đổi thứ tự node, đặt Start Node là node mong muốn.
2. Thực hành filter:
   - Gõ tên một tag vào ô filter → xác nhận chỉ hiện node có tag đó.
   - Xoá 1 node → xác nhận file gốc vẫn còn trên ổ đĩa.
3. Kiểm tra tooltip của 1 node: ghi lại **Node ID** – sau này sẽ dùng khi viết skin logic.
