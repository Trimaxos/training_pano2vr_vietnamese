# 5.2 – Các element cơ bản trong skin

## 1. Mục tiêu bài học

- Sau bài này, học viên sẽ:
  - Nhận diện được các element hay dùng trong skin.
  - Biết vai trò của từng loại element.
  - Bắt đầu xây dựng tư duy bố cục giao diện.

## 2. Các element thường gặp

| Element | Vai trò |
|---|---|
| Container | Nhóm nhiều phần tử lại với nhau |
| Image | Logo, icon, hình nền |
| Text | Tiêu đề, mô tả, nhãn nút |
| Button | Nút thao tác |
| Rectangle / Shape | Nền hoặc khối chia layout |
| Thumbnail list | Danh sách node dạng ảnh |

## 3. Container rất quan trọng

- Hầu hết skin tốt đều dùng `container` để nhóm phần tử.
- Lợi ích:
  - Dễ di chuyển cả cụm.
  - Dễ bật/tắt cùng nhau.
  - Dễ quản lý responsive hơn.

## 4. Text và Image

- `Text` nên dùng cho:
  - Tên tour.
  - Tên node hiện tại.
  - Tooltip đơn giản.

- `Image` nên dùng cho:
  - Logo thương hiệu.
  - Icon hotspot nếu skin điều khiển.
  - Nền texture nếu thật sự cần.

## 5. Button

- Nút thường gặp trong tour:
  - Home
  - Fullscreen
  - Toggle map
  - Toggle thumbnails
  - Next / Previous node

Nguyên tắc:
- Kích thước đủ bấm trên mobile.
- Icon dễ hiểu.
- Không nhồi quá nhiều nút trên màn hình.

## 6. Tổ chức element

- Đặt tên rõ ràng:
  - `btn-home`
  - `logo-brand`
  - `panel-menu`
  - `txt-node-title`

- Tránh để tên mặc định quá nhiều như:
  - `element 1`
  - `container 3`

## 9. Gợi ý bài tập

1. Mở một skin mẫu và đổi tên các element chính theo quy ước dễ đọc.
2. Tạo một container chứa logo và tiêu đề tour.
3. Tạo một nút đơn giản trong skin để làm quen với cấu trúc element.
